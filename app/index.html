--- The top toolbar contains three buttons on the right: **Add a Card**, **Help Dialog**, and **Download Card List**. On the left, there is a search filter field that will filter all card contents against its value. Just below the toolbar is the deck, where created cards are displayed and organized. If no cards are available, the ability to upload a previously saved card list becomes available. If desired, clicking the **Add a Card** button will start a new list. A saved card list is a simple plain text document using json syntax. --- ###Info Card An info card contains two fields: A **Card Title** and the **Card's Contents**. Mousing over the card's header will reveal two buttons: **Edit** and **Delete**. Clicking the **Edit** button will toggle the state of the card. Double clicking the **Delete** button will destroy the card. * **Card Title:** A simple identifier for individualizing a card. *If more than one card has the same title, a deletion of one of these cards will delete all of them.* * **Card Contents:** A text area that is parsed as markdown syntax. *"Markdown is not a replacement for HTML, or even close to it. Its syntax is very small, corresponding only to a very small subset of HTML tags. The idea is not to create a syntax that makes it easier to insert HTML tags. In my opinion, HTML tags are already easy to insert. The idea for Markdown is to make it easy to read, write, and edit prose. HTML is a publishing format; Markdown is a writing format. In order to modify the list order, simply drag a card to its desired postion, and the rest of the cards will reorder their layout.
